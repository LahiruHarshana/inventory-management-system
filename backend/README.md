# Inventory Management System Backend

A production-ready backend API for managing storage locations, inventory lifecycle, and controlled borrowing workflows.  
Built with Laravel 11 and PostgreSQL, this service emphasizes data integrity, concurrency safety, and traceable domain events for engineering-grade reliability.

## Key Features

- Token-based authentication using **Laravel Sanctum**.
- Role-based access control (**Admin** and **Staff**) for secured operational boundaries.
- Hierarchical storage model:
	- **Cupboards** as top-level storage units.
	- **Places** as child locations inside cupboards.
- Complete inventory management with full **CRUD** for items.
- Borrowing workflow with **pessimistic locking** to prevent:
	- race conditions during concurrent borrow operations,
	- negative stock balances,
	- inconsistent inventory state.
- Automatic audit trails through a custom **`Auditable` trait**:
	- hooks into Eloquent lifecycle events (`created`, `updated`, `deleted`),
	- stores old/new values in PostgreSQL **JSONB** for efficient history querying.
- Automated, always-current API documentation via **dedoc/scramble** at **`/docs/api`**.

## Installation & Setup

```bash
# 1) Clone the repository
git clone <your-repository-url>
cd inventory-management-system/backend

# 2) Install PHP dependencies
composer install

# 3) Create environment file
cp .env.example .env
```

Update your `.env` with database, Sanctum/session, and Cloudinary credentials:

```env
APP_NAME="Inventory Management System"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://127.0.0.1:8000

DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=inventory_db
DB_USERNAME=postgres
DB_PASSWORD=your_password

CLOUDINARY_URL=cloudinary://<api_key>:<api_secret>@<cloud_name>
# or equivalent individual Cloudinary keys depending on your config
```

Continue with application bootstrapping:

```bash
# 4) Generate app key
php artisan key:generate

# 5) Run database migrations
php artisan migrate

# 6) (Optional) Seed initial data
php artisan db:seed

# 7) Start development server
php artisan serve
```

Application will be available at:

- `http://127.0.0.1:8000`

## Architectural Decisions

### 1) Concurrency Control for Borrowing

The borrowing flow is intentionally designed to be safe under concurrent requests:

- Critical stock updates are wrapped in `DB::transaction(...)`.
- Target inventory rows are locked using `lockForUpdate()` before stock mutation.
- Availability checks and decrement operations are executed within the same transaction boundary.

This guarantees that parallel requests cannot overdraw stock, preventing race-condition bugs and preserving correct inventory balances at all times.

### 2) Audit Logging via Trait-Driven Domain Hooks

Audit history is implemented through a reusable custom `Auditable` trait:

- The trait subscribes to Eloquent model events (`created`, `updated`, `deleted`).
- It captures and persists actor/action context plus state transitions.
- Previous and current values are stored in PostgreSQL `JSONB` columns for flexible, query-friendly change tracking.

This approach centralizes cross-cutting auditing behavior, keeps model code clean, and creates an immutable operational timeline suitable for compliance, debugging, and reviewer verification.

## Security & Access Notes

- Sanctum-protected endpoints require authenticated tokens.
- Authorization logic enforces role-aware access for Admin and Staff operations.
- Sensitive credentials (`.env`) must never be committed.
